One design decision worth knowing about, since it's the thing that most often breaks a theme switcher: I used only classic tk widgets, no ttk anywhere. The ttk widgets don't accept bg/fg at all, so recoloring one raises a TclError at runtime — and because the inheritance between widget classes isn't always what you'd expect, a type-based theming loop can quietly hand a color option to a widget that can't take it. I sidestepped that entirely two ways: tk.OptionMenu instead of ttk.Combobox, and widgets registered by role as they're created rather than discovered by walking the widget tree. A widget can only ever receive the options it was registered for.
Testing found and fixed one real bug. exit_full_screen() originally only fired if the window manager reported fullscreen as active — which meant the size drop-down could get stuck displaying "Full Screen" while the window wasn't. It now checks the app's own state as well.
